Request Body and Response Formats

(1) Person format
{
"id": Number,
"firstName": String,
"lastName": String,
"street": String,
"city": String,
"zip": String,
"hobbies": String,
"phoneNo": ListString
}

(2) Person format (don’t provide ID, for POST)
{
"id": Number,
"firstName": String,
"lastName": String,
"email": String (email),
"street": String,
"additionalInfo": String,
"city": String,
"zip": String,
"hobbyName": String,
"hobbyDescription": String,
"phoneNumber": String,
"phoneDescription": String
}

Errors

(e) All errors are reported using this format (with the HTTP-status code matching the number)
{ status : statusCode, "msg": "Explains the problem" }

  • (e1) :{ status : 404, "msg": "No content found for this request" }
  • (e2) :{ status : 400, "msg": "Field ‘xxx’ is required" } (for example, no name provided)
